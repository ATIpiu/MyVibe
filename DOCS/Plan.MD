AI Coding Agent — 构建计划                             

 Context

 基于 Plan.MD 中对 Claude Code 架构的逆向工程，用 Python 构建一个类似的 CLI 编码代理。
 核心哲学：模型周围极简外壳，客户端执行工具，LLM 负责推理决策。
 技术栈：Python + Rich（替代 React/Ink）+ Anthropic SDK + YAML 配置。

 ---
 核心 Agentic 循环（coding_agent.py: run_turn）

 用户输入
   → state.append_user()
   → check_compress()（token > 92% 时压缩）
   → build_system_prompt()
   ↓
 while True:
   → llm_client.stream_chat(messages, system, tools, on_text=实时打印)
   → state.append_assistant(content_blocks)
   → state.update_usage() + session_manager.save()
   ↓
   stop_reason == "tool_use"?
     → handle_tool_calls()（ThreadPoolExecutor 并行，max_workers=4）
       → permission_manager.check() → 拒绝/确认/允许
       → ToolRegistry.instantiate(name).execute(**input)
       → _render_tool_call()（Rich Panel 展示）
     → state.append_tool_results()
     → 继续循环
   stop_reason == "end_turn"?
     → break，返回最终文本

 ---
 关键技术决策

 1. 工具唯一性检查：edit_file 的 old_string 出现次数 > 1 时立即返回错误，要求 LLM 提供更多上下文
 2. SSE JSON累积：_parse_stream 用 tool_input_buffer: dict[str, str] 按 tool_use_id 分别累积增量 JSON
 3. 流式渲染：on_text 直接 print(delta, end="", flush=True)，不用 Live 重绘（避免闪烁）
 4. 路径安全：所有文件工具调用 safe_resolve() 防路径遍历
 5. 并行工具：同路径的 edit_file/write_file 通过文件粒度锁串行化
